Overview

GraftM is a meta-omic tool that identifies and classifies marker genes in short read datasets (metagenomes and metatranscriptomes), as well as assembled contigs, whole genomes and protein sequences. It produces outputs including the sequences identified, a taxonomic/functional summary table, and krona plot.

GraftM is designed for speed and accuracy. It is able to find marker genes in a 200Mb of assembled metagenome in <20 sec, and compares favourably with similar tools in accuracy benchmarking.

Getting GraftM

Currently, GraftM has a few dependencies, and so simply using pip isn't really an option. Unfortunately that means that for the time being installation will have to be manual, although we're working on making the installation a little smoother so watch this space. The manual has more detailed instructions on how to install GraftM and a detailed description of command line options.

The source code for GraftM is also available on Github.

Using GraftM

GraftM was developed as a tool for the rapid analysis of short read metagenomic data. The idea was to develop a tool that could search and classify marker gene reads without depending on pairwise sequence comparisons, such as BLAST. The result is an easy-to-use tool which can be run on any marker gene of choice, in a fraction of time taken for other methods.

Currently, graftM can be used on 32 trusted marker genes, along with numerous metabolic genes that are phylogenetically consistent with 16S, such as McrA. In addition, GraftM can be used with other, functional genes to classify your sample based on functionality (e.g. PmoA)

Example Usage

Here is just a quick example of how to run GraftM, for impatient peoples. You can find some example data to use GraftM with here.

If you'd like to run GraftM with only a single file of reads, you'll use the --forward flag. Your run should something akin to this:

$ graftM graft --forward reads_f.fa --graftm_package mcrA_gpkg

The M suite

GraftM is part of the M(etagenomics) suite of tools being developed at the Australian Centre for Ecogenomics (ACE). We'd love for you to check out our tools:

  • GroopM - Recover (bin) population genomes from assembled metagenomes using differential coverage across multiple datasets.
  • CheckM - Assess the quality of genomes recovered from isolates, single cells or metagenomes using lineage specific marker sets.
  • BamM - Make and parse metagenomic BAM files.
